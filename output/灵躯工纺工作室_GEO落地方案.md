# 灵躯工纺工作室 GEO 落地方案

> 版本：1.0｜日期：2026-07-02｜执行周期：90 天  
> GEO 在本方案中指生成式引擎优化（Generative Engine Optimization）：让灵躯工纺在用户向 ChatGPT、Google AI、Bing/Copilot、DeepSeek、豆包、Kimi、腾讯元宝等提出行业问题时，更容易被正确理解、提及、引用，并最终形成有效询盘。

## 一、执行摘要

### 90 天业务目标

不是追求“AI 提到品牌”这一项虚荣指标，而是建立以下可验证链路：

**目标客户提出具体问题 → AI 能检索到灵躯工纺的专业答案与证据 → 品牌进入候选供应商名单 → 用户访问方案/案例页 → 提交机器人资料 → 形成打样机会。**

90 天建议目标：

| 目标 | 第 0 天基线 | 第 90 天目标 |
|---|---:|---:|
| 重点问题中的品牌提及率 | 首周实测 | ≥ 20% |
| 重点问题中的官网引用率 | 首周实测 | ≥ 10% |
| 已收录的核心页面 | 待 Search Console / Bing 核验 | 100% |
| 可公开、带数据的技术证据页 | 0–1 篇 | ≥ 6 篇 |
| 完整客户/联合验证案例 | 0 篇 | ≥ 3 篇 |
| 权威第三方有效提及 | 待核验 | ≥ 8 条 |
| GEO 来源有效询盘 | 待埋点 | ≥ 5 条/月 |

### 核心判断

现有网站已有首页、产品页、Roboskin 产品页、人形机器人方案页、FAQ、结构化数据、站点地图和 `llms.txt`，内容骨架已经形成。当前限制 AI 引用的关键不是“关键词数量不够”，而是：

1. **证据不足**：高弹、透气、可水洗、7–15 天打样等主张缺少测试条件、样品编号、数据表、照片或报告。
2. **案例不足**：缺少“客户问题—机器人结构—方案—测试—结果”的完整闭环。
3. **实体信号偏弱**：工作室、母公司、英文名、域名、地址、联系人在各平台尚未形成一致且可交叉验证的实体关系。
4. **内容过度集中在官网**：外部权威提及、产业媒体、合作机构、展会页面、专利/标准/论文等独立信号不足。
5. **页面意图仍偏宽**：FAQ 很全，但缺少可独立引用的深度答案页、对比页和采购决策页。
6. **技术细节有缺口**：需补全 canonical、更新时间、作者/审核人、站点级实体关系；并检查 `llms.txt` 的线上编码和可读性。

### GEO 战略一句话

**以“机器人针织软皮肤工程供应商”为唯一实体定位，用可复核数据和真实案例占领高采购意图问题，再用产业链第三方页面为官网结论背书。**

### 每篇“可引用页面”的固定结构

1. 题目直接对应一个问题，不写空泛标题。
2. 首屏用 40–80 字给出结论，允许 AI 独立摘录。
3. 明确适用对象和不适用边界。
4. 给出步骤、参数表或比较表。
5. 标注数据来源：内部测试/客户授权案例/公开标准/第三方资料。
6. 写清测试日期、环境、样品编号、测试方法和局限。
7. 使用原创图并配说明文字，不把关键信息只放在图片中。
8. 页面显示作者、技术审核人、首次发布和最后更新日期。
9. 加 3–5 个页面内 FAQ，但不为“刷结构化数据”而堆问答。
10. 末尾只放一个强 CTA：上传 3D/尺寸资料，获取技术评估。

## 五、证据工程：GEO 的主战场

### 建立“可公开参数表”

由技术负责人确定哪些参数可披露，不可用推测值。首批建议：

| 指标 | 测试表达要求 |
|---|---|
| 样件重量 | 覆盖部位、尺寸、材料组成、克重 |
| 双向延伸/回复 | 纵横向、测试方法、循环次数 |
| 关节疲劳 | 动作角度、频率、循环数、滑移/破损结果 |
| 透气或温升 | 环境温度、热源、对照组、测点和时长 |
| 耐磨 | 采用的方法、载荷、循环数、外观评级 |
| 洗涤 | 水温、程序、次数、尺寸变化和功能变化 |
| 接口避让 | 安全边界、开口加固方式、运动后偏移 |
| 打样周期 | 起算条件、标准版与功能版范围，不承诺无条件固定交期 |

### 最小可用案例模板

```text
案例标题：为某类人形机器人开发肩肘一体针织软皮肤
客户类型：整机厂 / 高校 / 展演团队（可匿名）
项目时间：YYYY-MM
输入条件：3D 文件格式、覆盖区域、关节角度、接口数量
原始问题：褶皱 / 滑移 / 散热 / 外观 / 维护
方案：材料、组织、弹性分区、固定和避让策略
迭代：首版发现什么，第二版改了什么
测试：环境、方法、循环数、关键数据
结果：达标项、未达标项、适用边界
客户确认：允许公开的评价或联合发布链接
```

### 内容真实性红线

- 没有测试，不写“不会影响散热”，改写为“可通过网眼分区与开孔降低包覆影响，实际温升需按本体测试”。
- 没有量产，不写“成熟量产能力”，改写为“支持打样及小批量，量产规模按设备与工艺评审”。
- 没有触觉样机数据，不写“实现高密度触觉”，改写为“支持导电纱线结构验证，传感性能取决于材料、采集电路与标定算法”。
- 引用 Figure、1X 等品牌只做客观工艺分析，使用官方公开资料并清晰标注来源，不暗示合作或可一比一复刻。
- 客户照片、机器人外观、3D 文件、测试数据须取得书面授权后公开。

## 六、技术 GEO 改造清单

### P0：7 天内完成

- [ ] 为每个 HTML 页面添加唯一 canonical，统一 `https`、主机名、尾斜杠规则。
- [ ] 核验正式线上域名；Cloudflare Pages 测试域名不宜长期作为品牌主域。
- [ ] 在 Google Search Console 与 Bing Webmaster Tools 验证站点并提交 sitemap。
- [ ] 确保 `robots.txt` 允许 Googlebot、Bingbot 和 OAI-SearchBot；同时检查 CDN/WAF 是否误拦截其官方 IP。
- [ ] 用 HTTP 状态和页面源代码核验所有 sitemap URL 均返回 200、无错误跳转。
- [ ] 检查线上 `llms.txt` 是否为 UTF-8 且中文无乱码；把它视作辅助说明，不视作排名捷径。
- [ ] 为现有结构化数据补全一致的 URL、logo、sameAs、母公司关系和联系方式。
- [ ] FAQ JSON-LD 与页面可见问答逐条保持一致；删除结构化数据中页面上不可见的答案。
- [ ] 在所有页显示清晰的更新时间；案例/知识页增加作者与技术审核人。
- [ ] 联系页增加可追踪表单，至少采集：机器人类型、覆盖部位、3D 资料情况、功能、数量、目标时间、来源。

### P1：30 天内完成

- [ ] 每个页面只服务一个主意图，避免首页、产品页、方案页重复同一段文字。
- [ ] 建立面包屑和上下文内链：“原理 → 方案 → 案例 → 评估”。
- [ ] 案例页使用 `Article` 或适配的页面类型，不编造不存在的 Schema 类型或评分。
- [ ] 产品/服务结构化数据只标注页面可见且真实的信息；没有公开价格则不填虚假 Offer。
- [ ] 图片文件名、alt 和图注描述真实内容；关键结构图提供文字说明。
- [ ] 为 PDF 测试报告制作同内容 HTML 摘要页，PDF 作为可下载证据，不让信息只困在附件里。
- [ ] 接入 IndexNow，在新增、更新、删除页面时主动通知支持的搜索引擎。
- [ ] 建立 404、重定向、重复标题、重复描述、孤儿页的月度检查。

### 对现有文件的具体建议

| 现有项 | 判断 | 动作 |
|---|---|---|
| `faq.html` | 覆盖广但部分结论过满 | 保留总 FAQ；把高价值问题拆成深度页，并修正无测试支撑的绝对表达 |
| `solutions-humanoid.html` | 场景完整 | 加真实结构图、限制条件、项目输入和案例内链 |
| `products-roboskin.html` | 可承接产品意图 | 增加规格边界、材料选项、测试与交付物 |
| 首页 Organization/Service JSON-LD | 有基础 | 补 `url` 一致性、`sameAs`、母公司关系、logo 绝对 URL，并与可见文案对齐 |
| `llms.txt` | 有主动意识 | 修复编码并精简为事实清单；不把其纳入 sitemap 的重点 URL |
| `sitemap.xml` | 已覆盖主要页面 | 增加真实 `lastmod`；新知识/案例页发布后自动纳入 |
| `robots.txt` | 当前允许全站 | 增加不是必要条件；重点检查 CDN/WAF 和线上响应 |

Google 明确表示，进入 AI Overviews / AI Mode 不需要特殊的 AI 文件或额外 Schema，基础 SEO、可抓取、文本可见、内链、页面体验以及结构化数据与可见内容一致仍是重点。ChatGPT Search 的官方要求则强调允许 OAI-SearchBot 抓取，并确保主机/CDN 不拦截其已公布 IP。因此，`llms.txt` 可以保留，但不应挤占案例与证据建设的优先级。

| P0 | 展会官方展商页/协会名录 | 标准公司简介、产品、官网链接 | 权威实体引用 |
| P1 | 机器人产业媒体 | 数据型案例、工艺解读，不发纯宣传稿 | 行业引用与发现 |
| P1 | 知乎 | 回答工程选型与采购问题，引用证据页 | 覆盖问题型搜索 |
| P1 | B 站/视频号 | 测试过程、关节运动、拆装、洗涤 | 视觉证据与品牌检索 |
| P2 | 小红书/抖音 | 展会样机、工艺过程、前后对比 | 品牌发现 |
| P2 | 百科类页面 | 达到收录条件后以可验证资料建设 | 实体消歧，不强求 |

### 每月最少产出

- 官网：2 篇证据型深度页 + 1 篇案例/测试记录。
- 视频：4 条 30–90 秒工程验证视频，官网同步图文摘要。
- 知乎/公众号：4 次专业回答或文章，链接到最匹配的证据页，不全部链首页。
- 第三方：至少促成 2 条真实的合作方/展会/媒体提及。

### 可执行的联合内容选题

1. 与机器人整机方联合发布“肩肘软皮肤 10,000 次运动测试”。
2. 与高校/材料方联合发布“导电纱线在曲面包覆中的信号稳定性验证”。
3. 与针织设备/纱线供应商发布“机器人非标曲面的针织组织选型”。
4. 在展会官方页面发布“机器人软皮肤现场拆装与关节运动演示”。

第三方文章必须由对方真实发布，避免批量软文站、链接农场、伪新闻稿和购买虚假问答。这些做法容易制造噪声，也无法形成可靠的品牌实体。

## 八、内容生产 SOP

### 每篇内容的 8 步流程

1. **选题**：从 30 个监测问题中选一个，确认对应决策阶段。
2. **访谈**：内容负责人用统一表单采访工程师，先拿事实再写文章。
3. **取证**：收集测试照片、原始数据、样品编号、测试条件和授权。
4. **成稿**：结论先行，写清适用边界，不做无证据承诺。
5. **技术审核**：由工程师对参数、术语、承诺逐项签字确认。
6. **发布**：HTML 正文、原创图、结构化数据、内链、单一 CTA 同步完成。
7. **分发**：同一事实改编为公众号、知乎、视频和销售资料，不全文复制。
8. **复测**：发布 14–30 天后复测收录、排名、AI 提及和询盘。

### 工程师采访提纲

- 这个问题在哪类机器人上最常发生？
- 现有方案为什么失败？失败表现是什么？
- 我们用了什么材料、组织和结构？
- 客户提供了哪些输入资料？
- 第一版有什么问题，后续如何修改？
- 测试条件、循环次数、结果是什么？
- 哪些数据可以公开？
- 方案在哪些情况下不适用？
- 如果客户咨询，第一步必须提供什么？

### AI 辅助写作规则

AI 可用于整理访谈、生成结构和改写多平台版本，不可生成测试数据、客户评价、合作关系、专利、认证、标准符合性或不存在的案例。技术负责人对最终稿承担事实审核责任。

## 十、监测与归因

### GEO 测试表字段

| 字段 | 示例 |
|---|---|
| 测试日期/平台/模型 | 2026-07-07 / ChatGPT Search / 当日版本 |
| 问题 | 国内有哪些做人形机器人软皮肤的供应商？ |
| 品牌是否提及 | 是/否 |
| 官网是否引用 | 是/否 |
| 引用 URL | 具体页面 |
| 提及位置 | 第 1/2/3 或正文 |
| 答案准确度 | 0–3 分 |
| 竞品/替代方案 | 名称与 URL |
| 截图/存档 | 文件链接 |
| 后续动作 | 补案例/修正事实/争取第三方提及 |

### 核心指标公式

- **品牌提及率** = 提及灵躯工纺的问题数 ÷ 测试问题总数。
- **官网引用率** = 引用灵躯官网的问题数 ÷ 测试问题总数。
- **引用准确率** = 准确引用次数 ÷ 全部引用次数。
- **核心页面收录率** = 已收录核心页 ÷ 已发布核心页。
- **有效询盘率** = 满足“有机器人项目 + 有明确场景/部位 + 有时间计划”的询盘 ÷ GEO 询盘。
- **GEO 商机贡献** = GEO 首触或助攻产生的有效商机金额/数量。

### 归因设置

- 表单增加“您从哪里了解到我们？”并提供“AI 助手/ChatGPT/DeepSeek/豆包/Kimi/元宝/搜索引擎”等选项。
- 所有平台链接使用 UTM，例如 `utm_source=zhihu&utm_medium=organic&utm_campaign=roboskin_geo`。
- 服务器日志按已公开的爬虫 IP/UA 规则识别抓取趋势，但不把 UA 单独视为可信身份。
- 每条销售线索必须记录首触、助攻、落地页和客户原始提问。
- AI 答案有随机性，固定问题、固定地区/语言、使用新会话，每轮至少测试两次，以趋势而非单次结果判断。

## 十三、决策原则

1. **先证据，后规模**：一篇带原始数据的案例，胜过十篇同义改写。
2. **先采购意图，后行业科普**：优先回答会推动技术评估的问题。
3. **一项事实，一个权威源**：官网是事实源，合作方/展会/媒体提供独立验证。
4. **承认边界更可信**：明确“不适用什么”有助于 AI 和客户正确理解能力。
5. **GEO 服从业务结果**：最终看有效询盘和商机，不以抓取量或单次提及自我安慰。
6. **不承诺排名**：生成式答案受索引、语境、模型、地区和时间影响，没有任何方法能保证固定提及。
